# 训练主循环 · 关键问题

---

## Q1：为何 RolloutManager 要先于 training models 创建？

**Explain：** `create_rollout_manager` 初始化 DataSource，根据 dataset 大小计算 `num_rollout_per_epoch`，供 save/eval periodic 与日志使用；并可能启动 SGLang 以推导 rollout 步数。

**Code：**

```python
# 来源：train.py L15-L17
    # need to initialize rollout manager first to calculate num_rollout
    rollout_manager, num_rollout_per_epoch = create_rollout_manager(args, pgs["rollout"])
```

---

## Q2：colocate 下为何 generate 前后要 offload/onload rollout？

**Explain：** 同一 GPU 上 Megatron 训练与 SGLang 推理不能同时占满显存；train 步把 rollout **offload 到 CPU**，train 完再 **onload 权重 → update_weights → onload KV** 准备下一轮 generate。

**Code：**

```python
# 来源：train.py L69-L92
        if args.offload_rollout:
            ray.get(rollout_manager.offload.remote())
        ...
        if args.offload_rollout:
            ray.get(rollout_manager.onload_weights.remote())
        actor_model.update_weights()
        if args.offload_rollout:
            ray.get(rollout_manager.onload_kv.remote())
```

**Comment：** `--colocate` 强制 `offload_train` + `offload_rollout`，见 [[03-Arguments-Ray-04-关键问题]]。

---

## Q3：async 为何禁止 colocate？

**Code：**

```python
# 来源：train_async.py L11
    assert not args.colocate, "Colocation is not supported for async training."
```

**Comment：** prefetch 假设 rollout GPU 在 train 期间持续 generate；colocate 的 offload 会破坏重叠语义。fully async 见 `examples/fully_async`。

---

## Q4：`update_weights_interval > 1` 有何影响？

**Explain：** 引擎 policy 权重滞后于 Megatron，形成 **staleness**；适合 decoupled 大规模 rollout，需算法侧容忍 off-policy。

**Code：**

```python
# 来源：train_async.py L65-L69
        if (rollout_id + 1) % args.update_weights_interval == 0:
            ...
            actor_model.update_weights()
```

**Comment：** 默认 `1`；与 Relax DCS 等系统级 async 不同层次。

---

## Q5：`num_critic_only_steps` 做什么？

**Explain：** PPO 常先训 value head 若干步再 joint 更新 policy。

**Code：**

```python
# 来源：train.py L72
        actor_trains_this_step = (not args.use_critic) or rollout_id >= args.num_critic_only_steps
```

**Comment：** GRPO 等 `use_critic=False` 时恒为 True。

---

## Q6：eval-only 怎么配？

**Explain：** `num_rollout == 0` 且配置 `eval_interval` / eval datasets；bootstrap 后跑一次 eval 即退出。

**Code：**

```python
# 来源：train.py L34-L36
    if args.num_rollout == 0 and args.eval_interval is not None:
        ray.get(rollout_manager.eval.remote(rollout_id=0))
```

## Q7：如何用测试理解 async？

**Explain：** `tests/test_qwen2.5_0.5B_async_short.py` 用 `train_async.py` 短配置断言 loss/grad 行为。

**Comment：** 读测试注释中的 `--update-weights-interval` 与 rollout batch 设置，对照 [[02-训练主循环-03-数据流与交互]] 流水线图。

## Q8：首次 update_weights 与循环内 update 区别？

**Explain：** 两者调用同一 API；首次在 **任何 generate 之前** 把 Megatron 初始权重（来自 ref_load）推到 SGLang，保证第一次 rollout 与训练权重一致。

**Code：**

```python
# 来源：train.py L25-L26
    # Always push actor weights to rollout once weights are loaded.
    actor_model.update_weights()
```

**Comment：** README 亦说明 hf-checkpoint 不必最新，因训练前总会 sync。

---

## Q9：save 何时包含 rollout dataset？

**Code：**

```python
# 来源：train.py L59-L60
        if args.rollout_global_dataset:
            ray.get(rollout_manager.save.remote(rollout_id))
```

**Comment：** 需启用 global dataset（默认未 `--disable-rollout-global-dataset`）。

| 只训 | `--debug-train-only` / `load_debug_rollout_data` | 无 generate |
