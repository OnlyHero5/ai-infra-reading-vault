# Alt-Rollout · 核心概念

---

## 1. Rollout 函数插拔契约

Slime 把「一次 rollout step 如何产出训练样本」抽象为可替换函数，由 CLI `--rollout-function-path` 指定。RolloutManager 在 init 时动态 import，每轮 `generate.remote(rollout_id)` 调用它。

**Explain：** 函数必须接受 `(args, rollout_id, data_source, evaluation=False)`。训练路径返回 `RolloutFnTrainOutput(samples=...)` 或等价结构；eval 路径返回 `RolloutFnEvalOutput(data=...)`。SFT / fully-async 等模块各自实现这一契约。

**Code：**

```python
# 来源：slime/rollout/sft_rollout.py L17-L28
def generate_rollout(args, rollout_id, data_buffer, evaluation=False):
    """An example to implement the generate_rollout function for an rule based rm rollout generation.

    Args:
        args: the whole args
        rollout_id: int, the id of the rollout, used for deterministic data generation
        data_buffer: the data buffer to store the generated samples
        evaluation: bool, whether the rollout is for evaluation or not

    Returns:
        list[Sample]: a list of samples generated by the rollout
    """
```

**Comment：**

- `data_buffer` / `data_source` 名称因实现而异，实质都是 [[11-DataSource-00-MOC]] 提供的采样入口。
- `evaluation=True` 时部分替代路径不支持（fully-async 直接 raise）。
- 返回值类型：新代码推荐 `RolloutFnTrainOutput`；SFT 仍返回扁平 `list`（框架兼容）。

## 2. 三层 hook 模型

| 层级 | CLI 参数 | 职责 | 本批示例 |
|------|----------|------|----------|
| 外层 rollout | `--rollout-function-path` | 整批 generate 编排 | fully-async、SFT、forge_load、sleep |
| 内层 generate | `--custom-generate-function-path` | 单 sample 调 SGLang | `generate_streaming` |
| 奖励 | `--custom-rm-path` + `--custom-reward-post-process-path` | RM + 后处理 | OPD `reward_func` / `post_process_rewards` |

**Explain：** 默认外层是 `sglang_rollout.generate_rollout`，内层 `generate` 发 HTTP 到 router。streaming 只替换内层；fully-async 替换外层但内层仍走 `generate_and_rm_group`。

**Code：**

```python
# 来源：slime/rollout/fully_async_rollout.py L8-L11
# Use with ``--rollout-function-path slime.rollout.fully_async_rollout.generate_rollout_fully_async``.
# Plug in per-sample logic via ``--custom-generate-function-path`` and
# per-sample reward via ``--custom-rm-path`` — the worker calls slime's stock
# :func:`generate_and_rm_group` which dispatches to those.
```

**Comment：**

- 组合灵活：fully-async + streaming generate + OPD RM 可同时启用。
- `generate_and_rm_group` 是内层 fan-out 与 group RM 的汇聚点（见批次 12）。

## 3. Fully-Async：解耦并发与 batch size

**动机：** 同步 rollout 每步必须等满 `rollout_batch_size` 组才返回；最慢样本拖住整步。Fully-async 用后台 worker 维持固定并发池，**跨 rollout 边界**持续从 buffer 取组、调用 `generate_and_rm_group`，主线程只收集已完成组直到 `target`。

**Explain：** 并发上限 = `sglang_server_concurrency × get_rollout_num_engines(args)`，与 `sglang_rollout` 内 semaphore 对齐。Worker 对 `ABORTED` 组回灌 buffer，不送入训练——权重更新后下一轮可重试。

**Code：**

```python
# 来源：slime/rollout/fully_async_rollout.py L53-L61
def _get_global_worker(args, data_buffer) -> AsyncRolloutWorker:
    global _global_worker
    with _worker_lock:
        if _global_worker is None or not _global_worker.worker_thread.is_alive():
            logger.info("starting fully-async rollout worker")
            _global_worker = AsyncRolloutWorker(
                args, data_buffer, concurrency=args.sglang_server_concurrency * get_rollout_num_engines(args)
            )
            _global_worker.start()
        return _global_worker
```

**Comment：**

- 全局单例 worker：rollout 之间队列保持温热，避免每步冷启动。
- Worker **不**处理 pause/weight-update 信号；各 `generate_and_rm_group` 任务自行检测 `GenerateState.aborted`。
- 与 `train_async.py` 的「训练当前步同时 generate 下一步」形成 **双重叠**（见 [[14-Alt-Rollout-03-数据流与交互]]）。

---

## 4. Streaming Generate：abort 友好的增量写入

**动机：** 默认 `generate` 等完整 JSON 返回；权重更新 abort 时依赖 `/abort_request` 回收 partial text。Streaming 每收到 SSE chunk 立即 `append_response_tokens` 到 `sample`，abort 切断流后 sample 上已有 coherent partial state。

**Explain：** 仅替换 HTTP 层；semaphore、dp_rank、partial_rollout buffer 仍由 `sglang_rollout` 管理。SGLang 默认 SSE 是 **cumulative**（每 chunk 含至今全部 token）。

**Code：**

```python
# 来源：slime/rollout/sglang_streaming_rollout.py L11-L14
# Wire it in as the per-sample generate function::
#     --rollout-function-path slime.rollout.sglang_rollout.generate_rollout \\
#     --custom-generate-function-path slime.rollout.sglang_streaming_rollout.generate_streaming
```

**Comment：**

- 若 SGLang 开启 `--incremental-streaming-output`，此处 delta 逻辑需改写。
- `state.aborted` 时 break SSE 循环，无 finish_reason 则标 `Sample.Status.ABORTED`。

## 5. SFT Rollout：不走 SGLang 的 generate

**动机：** 监督微调只需把 chat messages tokenize 并算 `loss_mask`，无需在线采样。SFT 复用 rollout 插槽，从 dataset 读 prompt，用 `MultiTurnLossMaskGenerator` 产出 tokens + mask，`reward=0`。

**Explain：** 全局 lazy 初始化 tokenizer / processor / mask generator；每个 sample 的 `prompt` 字段实际是 messages 列表。

**Code：**

```python
# 来源：slime/rollout/sft_rollout.py L44-L60
for i, sample in enumerate(samples):
    (sample,) = sample
    messages = sample.prompt
    tools = sample.metadata.get("tools", None)

    token_ids, loss_mask = MASK_GENERATOR.get_loss_mask(messages, tools=tools)
    # ...
    sample.tokens = token_ids
    sample.response_length = response_length
    sample.reward = 0
    sample.loss_mask = loss_mask[-response_length:]
```

**Comment：**

- 要求 `args.rollout_global_dataset`；不支持 evaluation。
- 示例脚本：`examples/geo3k_vlm/run_geo3k_vlm_sft.sh`。

## 6. On-Policy Distillation（OPD）

**动机：** 在 on-policy RL 框架内做教师-学生蒸馏。教师 server 返回 token 级 log-prob，存入 `sample.teacher_log_probs`，训练侧 OPD KL penalty 提供学习信号；标量 reward 可为 0。

**Explain：** `reward_func` 向 `args.rm_url`（教师 SGLang）POST；`post_process_rewards` 裁剪到 response 长度并写 `teacher_log_probs`。

**Code：**

```python
# 来源：slime/rollout/on_policy_distillation.py L58-L67
for sample, t_log_probs in zip(samples, teacher_log_probs, strict=False):
    sample.teacher_log_probs = t_log_probs

# Return scalar rewards for GRPO/PPO advantage estimator
# For pure on-policy distillation, we use 0.0 as the task reward.
scalar_rewards = [0.0] * len(samples)

return scalar_rewards, scalar_rewards
```

**Comment：**

- 与 [[21-Loss-Advantages-00-MOC]] 中 OPD KL 项配合。
- 非 rollout 替换——是 **RM hook**，外层仍用标准 sglang rollout。

## 7. Sleep 与 Forge Load：调试专用

| 模块 | 用途 | 是否启动 SGLang |
|------|------|-----------------|
| `sleep_rollout.sleep` | Profiling：Rollout 进程空转，便于单独 profile 训练 | 是（RolloutManager 正常 init） |
| `forge_load.generate_rollout` | 内存测试：从 `.pt` replay 样本，仍跑 weight update | 是 |
| `load-debug-rollout-data` | 纯训练 debug | **否**（skip_sglang） |

**Explain：** forge_load 故意 **不** 触发 `debug_train_only`，以便测量 colocate offload/onload 与长上下文真实显存。

**Code：**

```python
# 来源：slime/rollout/forge_load.py L19-L25
# Unlike --load-debug-rollout-data, this path does NOT set
# skip_sglang=True / debug_train_only=True (see
# slime/utils/arguments.py: skip_sglang computation in _pre_parse_mode and
# the debug_train_only flip when load_debug_rollout_data is set), so
# sglang servers, router, weight_update and the full colocate
# offload/onload dance still run.
```

**Comment：**

- **禁止** overwrite `sample.rollout_id`，否则 dp_schedule 分组断言失败。
- 路径支持 literal（复用 `0.pt`）或 `{rollout_id}` 模板。

## 8. 三种「异步」对比

| 维度 | sync `train.py` | `train_async.py` | + fully-async rollout |
|------|-----------------|------------------|------------------------|
| 训练/生成重叠 | 否 | 是（step N 训练 ∥ step N+1 generate） | 是 |
| 生成内部并发池 | sglang_rollout pendings | 同左 | 额外跨 step 后台 worker |
| colocate | 支持 | **不支持** | 不支持 |
| 典型场景 | 默认 GRPO/PPO | 隐藏 generate 延迟 | 高并发 + 长尾样本 |
